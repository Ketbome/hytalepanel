# Моди

Керуйте модами Hytale через панель з підтримкою Modtale та CurseForge.

## Провайдери Модів

Панель підтримує два репозиторії модів:

| Провайдер      | Веб-сайт                                               | Особливості                              |
| -------------- | ------------------------------------------------------ | ---------------------------------------- |
| **Modtale**    | [modtale.net](https://modtale.net)                     | Ком'юніті-репозиторій модів Hytale |
| **CurseForge** | [curseforge.com/hytale](https://curseforge.com/hytale) | Офіційна платформа модів Hytale (партнерство з Hypixel Studios)            |

Можна використовувати один або обидва провайдери. Панель автоматично визначає, з якого провайдера кожен мод.

## Налаштування

### API Ключ Modtale

1. Створіть обліковий запис на [modtale.net](https://modtale.net)
2. Отримайте API ключ у своєму профілі
3. Додайте до `.env`:

```env
MODTALE_API_KEY=ваш-api-ключ-тут
```

### API Ключ CurseForge

1. Перейдіть на [console.curseforge.com](https://console.curseforge.com)
2. Створіть проект та отримайте API ключ
3. Додайте до `.env`:

::: warning Важливо
API ключі CurseForge містять символи `$`. **Оберніть в одинарні лапки** щоб уникнути проблем:
:::

```env
CURSEFORGE_API_KEY='$2a$10$ваш-ключ-тут'
```

### Застосування Змін

Перезапустіть панель після додавання API ключів:

```bash
docker compose restart hytale-panel
```

## Використання Вкладки Модів

### Перегляд та Встановлення

1. Відкрийте панель та перейдіть на вкладку **Моди**
2. Натисніть **Огляд** щоб побачити доступні моди
3. Виберіть провайдера (Modtale або CurseForge) кнопками перемикання
4. Шукайте або переглядайте моди
5. Натисніть **Встановити** на потрібному моді

Панель показує візуальні індикатори для кожного провайдера:

- 🟢 Зелена точка = API налаштовано і працює
- 🔴 Червона точка = Невалідний API ключ
- ⚫ Сіра точка = Не налаштовано

### Встановлені Моди

Переключіться на перегляд **Встановлені** щоб побачити свої моди:

- Вмикати/вимикати моди
- Видаляти моди
- Переглядати деталі модів
- Бачити з якого провайдера кожен мод (значок Modtale або CurseForge)

### Перевірка Оновлень

Натисніть **Оновлення** щоб перевірити новіші версії встановлених модів:

- Працює з модами Modtale та CurseForge
- Перевіряє лише моди від налаштованих провайдерів
- Оновлення до останньої версії одним кліком

## Ручне Встановлення

Для модів, недоступних у репозиторіях:

1. Завантажте файли моду (`.jar` або `.zip`)
2. Використовуйте вкладку **Файли** для завантаження до папки `mods/`
3. Або розмістіть файли безпосередньо у `./data/panel/servers/{server-id}/server/mods/`

Ручні моди відображаються як "Локальні" у списку Встановлених.

## Увімкнення Модів

Сервер повинен знати де знаходяться моди. Налаштуйте через:

### На Сервер (Вкладка Конфіг)

1. Перейдіть на вкладку **Конфіг**
2. Встановіть **Додаткові аргументи** на: `--mods mods`
3. Збережіть та перезапустіть сервер

### Глобально (.env)

```env
SERVER_EXTRA_ARGS=--mods mods
```

## Власна Папка Модів

Підключіть власну папку модів у `docker-compose.yml`:

```yaml
services:
  hytale-server:
    volumes:
      - ./server:/opt/hytale
      - ./мої-моди:/opt/hytale/mods
```

## Команди Модів

Консольні команди сервера:

```
/mods list          # Список встановлених модів
/mods reload        # Перезавантажити всі моди
/mods enable <mod>  # Увімкнути мод
/mods disable <mod> # Вимкнути мод
```

## Вирішення Проблем

### "API ключ не налаштовано"

- Перевірте що у файлі `.env` правильний ключ
- Для CurseForge переконайтеся що ключ в одинарних лапках
- Перезапустіть панель після додавання ключів

### "Невалідний API ключ"

- Перевірте що ключ правильний (скопіюйте знову від провайдера)
- Ключі CurseForge починаються з `$2a$` - це нормально
- Перевірте чи не закінчився термін дії ключа

### Моди не завантажуються

1. Перевірте що `--mods mods` є в Додаткових аргументах або SERVER_EXTRA_ARGS
2. Перевірте що моди у правильній папці
3. Перегляньте логи сервера на помилки

### Мод CurseForge показує "Потрібно завантажити вручну"

Деякі моди CurseForge не дозволяють розповсюдження через API. Відвідайте сторінку моду для ручного завантаження.

### Проблеми з продуктивністю

- Моніторте використання RAM сервером
- Збільште `JAVA_XMX` якщо потрібно
- Деякі моди потребують більше ресурсів
